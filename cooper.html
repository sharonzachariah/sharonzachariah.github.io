Cooper is a conversational user interface for a guitar accessory called the loop pedal. It was created to increase accessibility and functionality for guitarists as they have busy hands.

RESTORING USER CONTROL

Loop pedals allow musicians to merge layers of music together to form a new song. However, they are tiny devices that don't have a lot of surface area for the amount of control this task demands.

 

By having a conversational interaction with the loop pedal, a guitarist can experience more flexibility and control over what the device can help them do.

    DESIGN STRATEGY

    Hands-free device

     

    When observing users interact with loop pedals, the most obvious pain point was when they needed to stop playing, put their guitar aside, and bend over to adjust the loop pedal. This series of events is conveniently avoided by introducing a conversational interface.

     

    Expert conversations

     

    Cooper is well equipped with the terminology and the techniques it needs to know to help its user. It is sensitive to what the guitarist may want to do at different stages of the interaction such as:

    - redoing a recording

    - muting certain layers

    - sharing a composition with band members or friends

     

    Dedicated channel for the guitar

     

    To ensure that the conversational interface doesn't take away from the quality of the music produced, I made sure to include a separate channel dedicated for Cooper to hear the guitar.

    Verbal and non-verbal communication

     

    During the prototyping phase, I learned that users found it easier to create loops by tapping on a button with their foot than to give a command to start the loop at the right beat. I made sure to include a standard button that most users are familiar with using.

    CREATING DIALOG FLOWS

    Dialog flows can be linear, or they can branch out to multiple responses based on the user's utterance. To account for the different use cases with Cooper, I used Dialogflow to delineate user intents, training phrases, responses, prompts and much more.

     

    Defining user intent

     

    To define the possible use cases a user would encounter with Cooper, I identified the common user intents associated with the interaction.

    Training phrases

     

    People use different words at different instances to convey their intent. To be prepared to handle all user utterances, a machine learning algorithm is trained on a sample set of phrases.

    Parameters help perform actions based on the user's preferences. In the following example, tempo and time_count are two parameters that help Cooper set the metronome.

    In the images below, the user wants to share his music with a friend. Cooper is designed to elicit any missing parameters from the query to successfully complete the request. A confirmation message at the end seals the deal.

    Building webhooks with messaging APIs

     

    I created a webhook in PHP and used Twilo's messaging API to bring Cooper's sharing capabilities to life. A user can now send and share their work with band members, friends, or other collaborators.
